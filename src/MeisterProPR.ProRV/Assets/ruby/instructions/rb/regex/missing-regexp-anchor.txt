Review focus: Missing regular expression anchor

What to look for:
- Regular expressions without anchors can be vulnerable to bypassing.
- Sanitizing untrusted input with regular expressions is a common technique. However, it is error-prone to match untrusted input against regular expressions without anchors such as`\A`or`\z`. Malicious input can bypass such security checks by embedding one of the allowed patterns in an unexpected location. Even if the matching is not done in a security-critical context, it may still cause undesirable behavior when the regular expression accidentally matches.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.
