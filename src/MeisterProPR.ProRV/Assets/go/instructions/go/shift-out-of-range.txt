Review focus: Shift out of range

What to look for:
- Shifting by more than the number of bits in the type of the left-hand side always yields 0 or -1.
- Shifting an integer value by more than the number of bits in its type always results in -1 for right-shifts of negative values and 0 for other shifts. Hence, such a shift expression is either redundant or indicates a logic mistake.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`ShiftOutOfRange.go`):
```
package main

func shift(base int32) int32 {
	return base << 40
}

var x1 = shift(1)
```
- Reference example (`ShiftOutOfRangeGood.go`):
```
package main

func shiftGood(base int64) int64 {
	return base << 40
}

var x2 = shiftGood(1)
```
