Review focus: Sizeof with side effects

What to look for:
- The sizeof operator should not be used on expressions that contain side effects.
- This rule finds`sizeof`expressions that have a parameter with side-effects.`sizeof`only uses the type of the parameter, so the parameter will not be evaluated. In the C99 standard, using`sizeof`on expressions with dynamic arrays may or may not evaluate the side-effect, so it is better to avoid it completely.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`AV Rule 166.cpp`):
```
int f(void){
	int i = 0;
	char arr[20];
	int size = sizeof(arr[i++]); //wrong: sizeof expression has side effect
	cout << i; //would output 0 instead of 1
}
```
