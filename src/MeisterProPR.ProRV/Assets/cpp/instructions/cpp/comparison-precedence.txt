Review focus: Unclear comparison precedence

What to look for:
- Using comparisons as operands of other comparisons is unusual in itself, and most readers will require parentheses to be sure of the precedence.
- This rule finds comparison expressions that use 2 or more comparison operators and are not completely parenthesized. It is best to fully parenthesize complex comparison expressions to explicitly define the order of the comparison operators.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`ComparisonPrecedence.cpp`):
```
void h() {
	int a, b, c;

	a < b != c; //parenthesize to explicitly define order of operators
	(a < b) < c; //correct: parenthesized to specify order
}
```
