Review focus: Register variables

What to look for:
- The register storage class specifier shall not be used.
- This query highlights variables with the`register`storage class specifier. Modern compilers are now capable of optimal register placement, and overriding it could lead to worse performance.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`AV Rule 140.cpp`):
```
int f() {
	//wrong: register storage specifier used
	register int i; //these register definitions can crowd out the
	                //variables x1 to x5 that are used in the complex
	                //operation in the inner loop, leading to slower
	                //performance
	register int j;

	for (i = 0; i < HUGE_NUM; i++) {
		for (j = 0; j < HUGE_NUM; j++) {
			int x1, x2, x3, x4, x5;
			//complex CPU-intensive operation that accesses
			//x1 to x5 a large number of times
		}
	}
}
```
