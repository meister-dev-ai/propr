Review focus: Local variable hides global variable

What to look for:
- A local variable or parameter that hides a global variable of the same name.
- This rule finds declarations of local variables or parameters that hide a global variable. Such declarations create variables with the same name but different scopes. This makes it difficult to know which variable is actually used in an expression.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`LocalVariableHidesGlobalVariable.cpp`):
```
int i = 10;

void f() {
	for (int i = 0; i < 10; i++) { //the loop counter hides the global variable i
		 ...
	}

	{
		int i = 12; //this variable hides the global variable i
		...
	}
}
```
