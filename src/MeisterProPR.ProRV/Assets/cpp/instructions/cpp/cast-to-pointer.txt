Review focus: Non-zero value cast to pointer

What to look for:
- A constant value other than zero is converted to a pointer type.
- This rule finds compile-time constants other than zero (and some common error markers, like`-1`and`0xdeadbeef`) which are (explicitly or implicitly) converted to a pointer type. This is a dangerous practice, since the meaning of the numerical value of pointers is platform dependent.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`NonzeroValueCastToPointer.cpp`):
```
void* p = (void*) 42; //Wrong: non-zero constant assigned to pointer. Is unportable 
                      //and will likely lead to a segfault.

void* p2 = NULL; //Correct: NULL (which is 0) assigned to a pointer
```
