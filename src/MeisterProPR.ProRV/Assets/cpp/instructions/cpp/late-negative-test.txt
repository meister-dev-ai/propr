Review focus: Pointer offset used before it is checked

What to look for:
- Accessing a pointer or array using an offset before checking if the value is positive may result in unexpected behavior.
- This query finds integer values that are first used to index an array and subsequently tested for being negative. If it is relevant to perform this test at all then it should happen before the indexing, not after. Otherwise, if the value is negative then the program will have failed before performing the test.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`LateNegativeTest.cpp`):
```
Record records[SIZE] = ...;

int f() {
	int recordIdx = 0;
	cin >> recordIdx;
	printRecord(&(records[recordIdx])); //incorrect: recordIdx may be negative here

	if (recordIdx >= 0) {
		processRecord(&(records[recordIdx])); //correct: index checked before use
	}
}
```
