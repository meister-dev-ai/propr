Review focus: Suspicious call to memset

What to look for:
- Use of memset where the size argument is computed as the size of some non-struct type.
- This rule finds calls to the standard library function`memset`where the third argument (specifying the number of bytes to set) does not appear to correspond with the first argument (the buffer to be written). Usually,`memset`is employed to initialize dynamically allocated arrays or structs. Since`memset`treats its first argument simply as an array of bytes, the third argument has to specify the size of the buffer in bytes. For an array, the size is the number of elements of the array multiplied by the size of one of its elements; for a`struct`, the size is just the size of the struct type. If`memset`is invoked with a third argument that is not constant and looks like neither of these two cases, there might be a mistake. This could cause a buffer overflow which could in turn cause a segfault or corrupt the contents of other variables in memory.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`SuspiciousCallToMemset.cpp`):
```
// correct, memset uses sizeof(type)
int is[10];
memset(is, 0, sizeof(is));
struct S s;
memset(&s, 0, sizeof(struct S));

// incorrect examples
struct T *t1 = (struct T*)malloc(sizeof(struct T));
struct T *t2 = (struct T*)malloc(sizeof(struct T));
// the size of the struct is probably intended
// but this takes the size of a pointer
memset(t2, 0, sizeof(t2));

// correct but discouraged, use sizeof(struct T) instead
memset(t1, 0, sizeof(*t2));
// correct, but it is preferred to do a direct assignment, i.e., t = 0;
memset(&t2, 0, sizeof(t2));
```
