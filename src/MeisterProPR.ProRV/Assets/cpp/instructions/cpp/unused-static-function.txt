Review focus: Unused static function

What to look for:
- A static function that is never called or accessed may be an indication that the code is incomplete or has a typo.
- This rule finds static functions with definitions that are never called or accessed. These unused functions should be removed to increase code readability, reduce object size and avoid misuse.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`UnusedStaticFunctions.cpp`):
```
//start of file
static void f() { //static function f() is unused in the file
    //...
}
static void g() {
    //...
}
void public_func() { //non-static function public_func is not called in file, 
                     //but could be visible in other files
    //...
    g(); //call to g()
    //...
}
//end of file
```
