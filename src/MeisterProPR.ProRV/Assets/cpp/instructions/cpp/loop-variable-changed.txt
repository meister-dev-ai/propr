Review focus: For loop variable changed in body

What to look for:
- Numeric variables being used within a for loop for iteration counting should not be modified in the body of the loop.
- This rule finds`for`loop variables that are being modified in the loop body.`for`loops should be reserved for simple iteration, use`while`loops for more complicated cases. Most developers assume that the`for`loop iteration statement is the only thing that changes the value of the loop counter, so changing it inside the loop body can lead to confusion.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`AV Rule 201.cpp`):
```
void f() {
	int i = 0;
	for (i = 0; i < 10; i++) {
		//...
		if (special_case) {
			i += 5; //Wrong: loop variable changed in body, which is contrary 
			        //to the usual expectation of for loop behavior. 
			        //Use a while loop instead.
			continue;
		}
	}
}
```
