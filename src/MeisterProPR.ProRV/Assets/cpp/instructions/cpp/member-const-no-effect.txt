Review focus: Constant return type on member

What to look for:
- A 'const' modifier on a member function return type is useless.
- This rule finds member functions with a superfluous`const`qualifier on their return type. This might be caused by confusion about how to declare a`const`method. In particular, when C++11 trailing return types are used, it becomes much easier to mis-declare a`const`method by putting`const`on the wrong side of`->`.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`ReturnConstTypeMember.cpp`):
```
struct S {
  int val;
  
  // The const has no effect here.
  auto getValIncorrect() -> const int {
    return val;
  }
  
  // Whereas here it does make a semantic difference.
  auto getValCorrect() const -> int {
    return val;
  }
};
```
