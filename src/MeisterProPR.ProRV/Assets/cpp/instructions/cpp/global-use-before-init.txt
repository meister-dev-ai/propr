Review focus: Global variable may be used before initialization

What to look for:
- Using an uninitialized variable may lead to undefined results.
- This rule finds calls to functions that use a global variable before the variable has been initialized. Not all compilers generate code that zero-out memory, especially when optimizations are enabled or the compiler is not compliant with the latest language standards. Accessing uninitialized memory will lead to undefined results.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`GlobalUseBeforeInit.cpp`):
```
int g_callCtr;

void initGlobals() {
	g_callCtr = 0;
}

int main(int argc, char* argv[]) {
	...
	cout << g_callCtr; //callCtr used before it is initialized
	initGlobals();
}
```
