Review focus: Function with too many parameters

What to look for:
- Functions with many parameters are hard to read and hard to use.
- Functions with many parameters are hard to understand and to use, and should be avoided.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`TooManyParameters.js`):
```
function sendRecord(firstName, lastName, dateOfBirth, streetAddress, postCode, city, country, email, website) {
	sendResponse({
		name: lastName + ', ' + firstName,
		DOB: dateOfBirth,
		address: streetAddress + '\n' + postCode + ' ' + city + '\n' + country,
		email: email,
		url: website
	});
}
```
- Reference example (`TooManyParametersGood.js`):
```
function sendRecord(record) {
	sendResponse({
		name: record.lastName + ', ' + record.firstName,
		DOB: record.dateOfBirth,
		address: record.streetAddress + '\n'
		       + record.postCode + ' ' + record.city + '\n'
		       + record.country,
		email: record.email,
		url: record.website
	});
}
```
