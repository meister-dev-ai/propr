Review focus: Setter ignores its parameter

What to look for:
- A setter function can silently ignore the new value that the property is meant to be set to, but this may result in unexpected behavior and could indicate a bug.
- Property setter functions that silently ignore their parameter make code hard to understand and could indicate an underlying bug.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`SetterIgnoresParameter.js`):
```
function Point(x, y) {
	return {
		get x() { return x; },
		set x(_x) { x = _x|0; },
		get y() { return y; },
		set y(_y) { y = _x|0; }
	};
}
```
- Reference example (`SetterIgnoresParameterGood.js`):
```
function Point(x, y) {
	return {
		get x() { return x; },
		set x(_x) { x = _x|0; },
		get y() { return y; },
		set y(_y) { y = _y|0; }
	};
}
```
