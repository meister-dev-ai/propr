Review focus: Jump from finally

What to look for:
- Avoid using unstructured control flow statements (return, continue, or break) inside a 'finally' block.
- A`finally`block is normally used to perform cleanup operations at the end of a`try`statement; it is run no matter which way the associated`try`statement terminates: successful termination, a`return`statement, or an exception being thrown. If a`finally`block contains a`return`statement or a`break`or`continue`statement referring to a loop outside the`try`statement, this statement will jump out of the`finally`block, thus overriding any return value or exception throw arising from the`try`statement. This is often unintentional, and in any case makes the code hard to read.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`JumpFromFinally.js`):
```
var resource;
try {
	resource = acquire();
	if (someCond())
		throw new Error();
	performAction(resource);
} finally {
	resource.release();
	if (someOtherCond())
		return true;
}
```
- Reference example (`JumpFromFinallyGood.js`):
```
var resource;
try {
	resource = acquire();
	if (someCond()) {
		if (someOtherCond())
			return true;
		else
			throw new Error();
	}
	performAction(resource);
	if (someOtherCond())
		return true;
} finally {
	resource.release();
}
```
