Review focus: Use of eval

What to look for:
- The 'eval' function and the 'Function' constructor execute strings as code.
- The built-in`eval`function and the`Function`constructor allow executing arbitrary strings as JavaScript code. This is a dangerous feature, since this code has the same access privileges as any other code, so great care has to be taken to ensure that malicious code is not accidentally executed this way. Using this feature also hampers static checking and program comprehension. In many cases, better alternatives are available and should be used instead.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`Eval.js`):
```
function Point(x, y) {
	this.x = x;
	this.y = y;
}

["x", "y"].forEach(function(p) {
	eval("Point.prototype.get_" + p + " = function() {" +
	     "  return this." + p + ";" +
	     "}");
	eval("Point.prototype.set_" + p + " = function(v) {" +
	     "  if (typeof v !== 'number')" +
	     "    throw Error('number expected');" +
	     "  this." + p + " = v;" +
	     "}");
});
```

- Reference example (`EvalGood.js`):
```
function Point(x, y) {
	this.x = x;
	this.y = y;
}

["x", "y"].forEach(function(p) {
	Point.prototype["get_" + p] = function() {
		return this[p];
	};
	Point.prototype["set_" + p] = function(v) {
		if (typeof v !== 'number')
			throw Error('number expected');
		this[p] = v;
	};
});
```
