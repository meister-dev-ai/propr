Review focus: Loop iteration skipped due to shifting

What to look for:
- Removing elements from an array while iterating over it can cause the loop to skip over some elements, unless the loop index is decremented accordingly.
- Items can be removed from an array using the`splice`method, but when doing so, all subsequent items will be shifted to a lower index. If this is done while iterating over the array, the shifting may cause the loop to skip over the element immediately after the removed element.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`LoopIterationSkippedDueToShifting.js`):
```
function removePathTraversal(path) {
  let parts = path.split('/');
  for (let i = 0; i < parts.length; ++i) {
    if (parts[i] === '..') {
      parts.splice(i, 1);
    }
  }
  return path.join('/');
}
```
- Reference example (`LoopIterationSkippedDueToShiftingGood.js`):
```
function removePathTraversal(path) {
  let parts = path.split('/');
  for (let i = 0; i < parts.length; ++i) {
    if (parts[i] === '..') {
      parts.splice(i, 1);
      --i; // adjust for array shift
    }
  }
  return path.join('/');
}
```
- Reference example (`LoopIterationSkippedDueToShiftingGoodFilter.js`):
```
function removePathTraversal(path) {
  return path.split('/').filter(part => part !== '..').join('/');
}
```
