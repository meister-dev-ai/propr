Review focus: Conflicting HTML element attributes

What to look for:
- If an HTML element has two attributes with the same name but different values, its behavior may be browser-dependent.
- According to the HTML5 standard, an HTML element must not have two or more attributes with the same name. Elements that do not conform to this restriction may be interpreted differently by different browsers.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`ConflictingAttributes.html`):
```
<a href="http://semmle.com" href="https://semmle.com">Semmle</a>
```
- Reference example (`ConflictingAttributesGood.html`):
```
<a href="https://semmle.com">Semmle</a>
```
