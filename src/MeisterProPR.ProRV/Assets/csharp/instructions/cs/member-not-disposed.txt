Review focus: Missing Dispose call

What to look for:
- Classes that implement 'IDisposable' and have members of 'IDisposable' type should dispose those members in their 'Dispose()' method.
- Classes that implement`IDisposable`and have members of`IDisposable`type should dispose those members in their`Dispose()`method.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`MissingDisposeCallBad.cs`):
```
using System;
using System.IO;

class Bad : IDisposable
{
    private FileStream stream1 = new FileStream("a.txt", FileMode.Open);
    private FileStream stream2 = new FileStream("b.txt", FileMode.Open);

    public void Dispose()
    {
        stream1.Dispose();
    }
}
```
- Reference example (`MissingDisposeCallGood.cs`):
```
using System;
using System.IO;

class Good : IDisposable
{
    private FileStream stream1 = new FileStream("a.txt", FileMode.Open);
    private FileStream stream2 = new FileStream("b.txt", FileMode.Open);

    public void Dispose()
    {
        stream1.Dispose();
        stream2.Dispose();
    }
}
```
