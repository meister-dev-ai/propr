Review focus: Cast to same type

What to look for:
- A cast to the same type as the original expression is always redundant.
- Casting an expression to the type that it already has serves no purpose and clutters the code. It indicates confusion about the type of the expression, or that the code has been partially refactored. This query applies to both the`()`operator and the`as`operator.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`UselessCastToSelf.cs`):
```
Dictionary<string, int> properties;

public int Size
{
    get { return (int)properties["Size"]; }
}
```
- Reference example (`UselessCastToSelfFix.cs`):
```
Dictionary<string, int> properties;

public int Size
{
    get { return properties["Size"]; }
}
```
