Review focus: Missing Dispose method

What to look for:
- Classes that implement 'IDisposable' and have members of 'IDisposable' type should also declare/override 'Dispose()'.
- Classes that implement`IDisposable`and have members of`IDisposable`type should also declare/override`Dispose()`.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`MissingDisposeMethodBad.cs`):
```
using System;
using System.IO;

class BadBase : IDisposable
{
    private FileStream stream1 = new FileStream("a.txt", FileMode.Open);

    public virtual void Dispose()
    {
        stream1.Dispose();
    }
}

class Bad : BadBase
{
    private FileStream stream2 = new FileStream("b.txt", FileMode.Open);
}
```
- Reference example (`MissingDisposeMethodGood.cs`):
```
using System;
using System.IO;

class GoodBase : IDisposable
{
    private FileStream stream1 = new FileStream("a.txt", FileMode.Open);

    public virtual void Dispose()
    {
        stream1.Dispose();
    }
}

class Good : BadBase
{
    private FileStream stream2 = new FileStream("b.txt", FileMode.Open);

    public override void Dispose()
    {
        base.Dispose();
        stream2.Dispose();
    }
}
```
