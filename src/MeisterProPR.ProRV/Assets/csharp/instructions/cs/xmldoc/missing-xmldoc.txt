Review focus: Missing documentation for public class or method

What to look for:
- A public class, method or constructor that does not have a documentation comment affects maintainability.
- Public APIs (including methods, constructors, classes, structs and interfaces) should be documented to help make the API easier to understand and maintain. For the purpose of code maintainability, it is also advisable to document non-public APIs.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`XmldocExample.cs`):
```
using System;
using System.Threading;

/// <summary>
///   A minimal threadsafe counter.
/// </summary>
class AtomicCounter
{
    /// <summary>
    ///   The current value of the counter.
    /// </summary>
    int currentValue = 0;

    /// <summary>
    ///   Increments the value of the counter.
    /// </summary>
    ///
    /// <param name="incrementBy">The amount to increment.</param>
    /// <exception cref="System.OverflowException">If the counter would overflow.</exception>
    /// <returns>The new value of the counter.</returns>
    ///
    /// <remarks>This method is threadsafe.</remarks>
    public int Increment(int incrementBy = 1)
    {
        int oldValue, newValue;
        do
        {
            oldValue = currentValue;
            newValue = oldValue + incrementBy;
            if (newValue < 0) throw new OverflowException("Counter value is out of range");
        }
        while (oldValue != Interlocked.CompareExchange(ref currentValue, newValue, oldValue));
        return newValue;
    }
}
```
