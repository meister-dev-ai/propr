Review focus: Container contents are never initialized

What to look for:
- Querying the contents of a collection or map that is never initialized is not normally useful.
- Code that queries the contents of a collection (such as`ContainsKey`or`Count`) is invoked on an object that is known to be empty. Such queries do not return interesting results, and may indicate missing code or a logic error.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`ReadOnlyContainer.cs`):
```
class Calendar
{
    IList<string> daysOfWeek = new List<string>();

    public string dayName(int day)
    {
        return daysOfWeek[day - 1];
    }
}
```
- Reference example (`ReadOnlyContainerFix.cs`):
