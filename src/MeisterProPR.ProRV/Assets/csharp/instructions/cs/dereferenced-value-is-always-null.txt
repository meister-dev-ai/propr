Review focus: Dereferenced variable is always null

What to look for:
- Dereferencing a variable whose value is 'null' causes a 'NullReferenceException'.
- If a variable is dereferenced, for example as the qualifier in a method call, and the variable has a`null`value on all possible execution paths leading to the dereferencing, the dereferencing is guaranteed to result in a`NullReferenceException`.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`NullAlwaysBad.cs`):
```
using System;

namespace NullAlways
{
    class Bad
    {
        void DoPrint(string s)
        {
            if (s != null || s.Length > 0)
                Console.WriteLine(s);
        }
    }
}
```
- Reference example (`NullAlwaysGood.cs`):
```
using System;

namespace NullAlways
{
    class Good
    {
        void DoPrint(string s)
        {
            if (s != null && s.Length > 0)
                Console.WriteLine(s);
        }
    }
}
```
