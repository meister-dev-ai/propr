Review focus: Text has not been internationalized

What to look for:
- Including text directly on a page makes it more difficult to internationalize the website.
- Some websites are internationalized, meaning they are implemented in a way that supports translation and adaption into multiple natural languages. For such a website, it is an error to embed English text directly into the page.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`NonInternationalizedTextBad.aspx`):
```
<%@ Page Language="C#" %>

<html>
<body>
<p>Amount: <%= Amount %></p>
</body>
</html>
```
- Reference example (`NonInternationalizedTextGood.aspx`):
```
<%@ Page Language="C#" %>

<html>
<body>
<p><%= I18n.getMessage("Amount") %>: <%= Amount %></p>
</body>
</html>
```
