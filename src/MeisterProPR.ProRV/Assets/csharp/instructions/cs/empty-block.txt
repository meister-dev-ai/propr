Review focus: Empty branch of conditional, or empty loop body

What to look for:
- Empty blocks that occur as a branch of a conditional or as a loop body may indicate badly-maintained code or a bug due to an unhandled case.
- Empty blocks or blocks with only code that has been commented out should be reviewed. It is common to find commented-out code in an empty block. Commented-out code is discouraged because it is a source of defects and maintainability issues.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`EmptyBlock.cs`):
```
using System;

class EmptyBlock
{
    static void Main(string[] args)
    {
        int i = 23;

        if (i == 42)
        {
            // TODO: Currently still research problems - professors aware.
#if false
            AchieveEnlightenment();
            CureCancer();
            world.SetPeaceful(true);
#endif
        }
        else
        {
            Console.WriteLine("Welcome to this meeting of the Realist Society. Join us!");
        }

        // After removing the empty block:
        if (i != 42)
        {
            Console.WriteLine("Welcome to this meeting of the Realist Society. Join us!");
        }
    }
}
```
