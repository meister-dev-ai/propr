Review focus: Useless 'is' before 'as'

What to look for:
- The C# 'as' operator performs a type test - there is no need to precede it with an 'is'.
- The C#`as`operator is equivalent to testing if an object is of a specific type with`is`and then performing a cast or returning null depending on the result. Performing an`is`check before calling`as`is completely redundant because an`is`check is performed twice.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`UselessIsBeforeAs.cs`):
```
class UselessIsBeforeAs
{
    class Square
    {
        public int width;
    }
    class Rectangle : Square
    {
        public int height;
    }
    public static void Main(string[] args)
    {
        Square a = new Rectangle();
        if (a is Rectangle)
        {
            Rectangle r = a as Rectangle;
            Console.WriteLine(r.height);
        }
    }
}
```
- Reference example (`UselessIsBeforeAsFix.cs`):
```
class UselessIsBeforeAs
{
    class Square
    {
        public int width;
    }
    class Rectangle : Square
    {
        public int height;
    }
    public static void Main(string[] args)
    {
        Square a = new Rectangle();
        Rectangle r = a as Rectangle;
        if (r != null)
        {
            Console.WriteLine(r.height);
        }
    }
}
```
