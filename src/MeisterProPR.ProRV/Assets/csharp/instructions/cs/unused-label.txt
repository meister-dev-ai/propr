Review focus: Unused label

What to look for:
- An unused label serves no purpose and should be removed.
- An unused label serves no purpose and clutters the source code. It could be an indication that the code is incomplete, or that the code contains a bug where a`goto`statement uses the wrong label.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`UnusedLabel.cs`):
```
static void Main(string[] args)
{
    using (var file = File.Open("values.xml", FileMode.Create))
    using (var stream = new StreamWriter(file))
    {
        stream.WriteLine("<values>");
        foreach (var arg in args)
        {
            uint value;
            if (UInt32.TryParse(arg, out value))
                stream.WriteLine("    <value>{0}</value>", value);
            else
                goto Finish;  // Should be goto Error
        }
        goto Finish;
        Error:  // BAD: Unused label
        Console.WriteLine("Error: All arguments must be non-negative integers");
        Finish:
        stream.WriteLine("</values>");
    }
}
```
