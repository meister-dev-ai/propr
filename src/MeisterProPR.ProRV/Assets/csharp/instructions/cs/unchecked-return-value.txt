Review focus: Unchecked return value

What to look for:
- If most of the calls to a method use the return value of that method, the calls that do not check the return value may be mistakes.
- Ignoring a method's return value can be a security risk and a common source of defects. If an attacker can force the method to fail, the subsequent program logic could lead to a vulnerability because the program will be running in an unexpected state. This rule checks that the return value of standard library methods like`System.IO.Stream.Read(...)`is always used. Furthermore, it identifies any calls to methods that ignore the return value despite it being frequently used elsewhere. That is, if more than 90% of the total calls to a particular method use its return value, all other calls that discard its return value are flagged as suspicious.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`UncheckedReturnValueBad.cs`):
```
using System;
using System.IO;

class Bad
{
    public bool DoPrint(string s) => true;

    public void IgnoreOne()
    {
        if (DoPrint("A"))
            Console.WriteLine("A");
        if (DoPrint("B"))
            Console.WriteLine("B");
        if (DoPrint("C"))
            Console.WriteLine("C");
        if (DoPrint("D"))
            Console.WriteLine("D");
        if (DoPrint("E"))
            Console.WriteLine("E");
        if (DoPrint("F"))
            Console.WriteLine("F");
        if (DoPrint("G"))
            Console.WriteLine("G");
        if (DoPrint("H"))
            Console.WriteLine("H");
        if (DoPrint("I"))
            Console.WriteLine("I");

        DoPrint("J");
    }

    void IgnoreRead(string path)
    {
        var file = new byte[10];
        using (var f = new FileStream(path, FileMode.Open))
            f.Read(file, 0, file.Length);
    }
}
```
