Review focus: Missing enum case in switch

What to look for:
- A 'switch' statement that is based on an 'enum' type and does not have cases for all the 'enum' constants is usually a coding mistake.
- A`switch`statement that is based on a variable with an`enum`type should either have a default case or handle all possible constants of that`enum`type. Handling all but one or two`enum`constants is usually a coding mistake.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.
