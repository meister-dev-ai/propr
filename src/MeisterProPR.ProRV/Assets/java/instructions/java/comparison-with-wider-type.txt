Review focus: Comparison of narrow type with wide type in loop condition

What to look for:
- Comparisons between types of different widths in a loop condition can cause the loop to behave unexpectedly.
- In a loop condition, comparison of a value of a narrow type with a value of a wide type may always evaluate to`true`if the wider value is sufficiently large (or small). This is because the narrower value may overflow. This can lead to an infinite loop.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
