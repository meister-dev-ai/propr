Review focus: Partial path traversal vulnerability

What to look for:
- A prefix used to check that a canonicalised path falls within another must be slash-terminated.
- A common way to check that a user-supplied path`SUBDIR`falls inside a directory`DIR`is to use`getCanonicalPath()`to remove any path-traversal elements and then check that`DIR`is a prefix. However, if`DIR`is not slash-terminated, this can unexpectedly allow access to siblings of`DIR`. See also`java/partial-path-traversal-from-remote`, which is similar to this query but only flags instances with evidence of remote exploitability.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.
