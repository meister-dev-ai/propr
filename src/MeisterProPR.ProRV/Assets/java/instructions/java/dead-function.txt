Review focus: Dead method

What to look for:
- Dead methods add unnecessary complexity.
- Methods that are never called at runtime are redundant and should be removed. Methods are considered dead if at runtime they are never called, either directly, by a method call, or indirectly, through a framework or use of reflection. Any method which is not dead is considered to be "live". The results can include methods, constructors and initializers. Initializers come in two forms, instance initializers and static initializers. For each class there will be at most one dead initializer of each type, representing all the initialization of that type in the class.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.
