Review focus: Inefficient empty string test

What to look for:
- Checking a string for equality with an empty string is inefficient.
- When checking whether a string`s`is empty, perhaps the most obvious solution is to write something like`s.equals("")`(or`"".equals(s)`). However, this actually carries a fairly significant overhead, because`String.equals`performs a number of type tests and conversions before starting to compare the content of the strings.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
