Review focus: Unsafe certificate trust

What to look for:
- SSLSocket/SSLEngine ignores all SSL certificate validation errors when establishing an HTTPS connection, thereby making the app vulnerable to man-in-the-middle attacks.
- Java offers two mechanisms for SSL authentication - trust manager and hostname verifier (the later is checked by the`java/insecure-hostname-verifier`query). The trust manager validates the peer's certificate chain while hostname verification establishes that the hostname in the URL matches the hostname in the server's identification. When`SSLSocket`or`SSLEngine`are created without a secure`setEndpointIdentificationAlgorithm`, hostname verification is disabled by default. This query checks whether`setEndpointIdentificationAlgorithm`is missing, thereby making the application vulnerable to man-in-the-middle attacks. The query also covers insecure configurations of`com.rabbitmq.client.ConnectionFactory`.
