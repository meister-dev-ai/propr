Review focus: Insecure basic authentication

What to look for:
- Basic authentication only obfuscates username/password in Base64 encoding, which can be easily recognized and reversed.
- Basic authentication only obfuscates usernames and passwords in Base64 encoding, which can be easily recognized and reversed, thus it must not be transmitted over the cleartext HTTP channel. Transmitting sensitive information without using HTTPS makes the data vulnerable to packet sniffing.

- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:

- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
