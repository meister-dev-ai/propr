Review focus: EJB uses non-final static field

What to look for:
- An EJB should not make use of non-final static fields, since a consistent state of such fields is not guaranteed if an EJB instance is distributed across multiple JVMs.
- The Enterprise JavaBeans 3.0 core specification, Section 21.1.2, states: An enterprise bean must not use read/write static fields. Using read-only static fields is allowed. Therefore, it is recommended that all static fields in the enterprise bean class be declared as final. This rule is required to ensure consistent runtime semantics because while some EJB containers may use a single JVM to execute all enterprise bean's instances, others may distribute the instances across multiple JVMs.
