Review focus: Detect JHipster Generator Vulnerability CVE-2019-16303

What to look for:
- Using a vulnerable version of JHipster to generate random numbers makes it easier for attackers to take over accounts.
- This query detects instances of`RandomUtil.java`that were generated by a JHipster version that is vulnerable to CVE-2019-16303. If an app uses`RandomUtil.java`generated by a vulnerable version of JHipster, attackers can request a password reset token and use this to predict the value of future reset tokens generated by this server. Using this information, they can create a reset link that allows them to take over any account. This vulnerability has a CVSS v3.0 Base Score of 9.8/10 .
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.
