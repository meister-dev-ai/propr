Review focus: Use of an undefined global variable

What to look for:
- Using a global variable before it is initialized causes an exception.
- This global variable may not be defined. If this code is executed and the variable is undefined then a`NameError`will occur.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`UndefinedGlobal.py`):
```
import math

angle = 0.01

sin(angle)       # NameError: name 'sin' is not defined (function imported from 'math')

math.sin(angle)  # 'sin' function now correctly defined

math.tan(angel)  # NameError: name 'angel' not defined (typographic error)

math.tan(angle)  # Global variable now correctly defined
```
