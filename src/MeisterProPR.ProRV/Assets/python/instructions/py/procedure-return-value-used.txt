Review focus: Use of the return value of a procedure

What to look for:
- The return value of a procedure (a function that does not return a value) is used.
- All functions in Python return a value. If a function has no`return`statements or none of the`return`statements return a value then the function will return`None`. However, this value has no meaning and should be ignored. Using the return value of such a 'procedure' is confusing to the reader as it suggests that the value is significant.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`UseImplicitNoneReturnValue.py`):
```
import sys

def my_print(*args):
    print (args)

def main():
    err = my_print(sys.argv)
    if err:
        sys.exit(err)

    my_print(sys.argv)
```
