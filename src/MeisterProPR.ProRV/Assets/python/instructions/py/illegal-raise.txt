Review focus: Illegal raise

What to look for:
- Raising a non-exception object or type will result in a TypeError being raised instead.
- If the object raised is not a legal Exception class or an instance of one, then a`TypeError`will be raised instead. Legal exception classes are:
- Any old-style classes (Python 2 only)
- Any subclass of the builtin class`BaseException`However, it recommended that you only use subclasses of the builtin class`Exception`(which is itself a subclass of`BaseException`).
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`IllegalRaise.py`):
```
#Cannot raise an int, even if we want to
def raise_int():
    #Will raise a TypeError
    raise 4
```
