Review focus: Explicit export is not defined

What to look for:
- Including an undefined attribute in `__all__` causes an exception when the module is imported using '*'
- When a module is imported using`import *`, all attributes listed in`__all__`are imported. If`__all__`includes attributes that are not defined in the module then an exception is triggered. This usually indicates a typographic error in the attributes in`__all__`or in the name of the object.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`UndefinedExport.py`):
```
__all__ = ['spamm', 'troll', 'paywall']

def spam(): return 'Spam'
def troll(): return 'Troll'
def paywall(): return 'Pay wall'
```
