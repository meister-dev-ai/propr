Review focus: Use of a broken or weak cryptographic hashing algorithm on sensitive data

What to look for:
- Using broken or weak cryptographic hashing algorithms can compromise security.
- Using a broken or weak cryptographic hash function can leave data vulnerable, and should not be used in security related code. A strong cryptographic hash function should be resistant to:
- pre-image attacks: if you know a hash value`h(x)`, you should not be able to easily find the input`x`.
- collision attacks: if you know a hash value`h(x)`, you should not be able to easily find a different input`y`with the same hash value`h(x) = h(y)`. In cases with a limited input space, such as for passwords, the hash function also needs to be computationally expensive to be resistant to brute-force attacks. Passwords should also have an unique salt applied before hashing, but that is not considered by this query. As an example, both MD5 and SHA-1 are known to be vulnerable to collision attacks. Since it's OK to use a weak cryptographic hash function in a non-security context, this query only alerts when these are used to hash sensitive data (such as passwords, certificates, usernames). Use of broken or weak cryptographic algorithms that are not hashing algorithms, is handled by the`py/weak-cryptographic-algorithm`query.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is path-sensitive. Keep it only when the diff suggests a plausible source-to-sink or input-to-operation path worth deeper inspection.

Reference examples:
- Reference example (`weak_certificate_hashing.py`):
```
import hashlib

def certificate_matches_known_hash_bad(certificate, known_hash):
    hash = hashlib.md5(certificate).hexdigest() # BAD
    return hash == known_hash

def certificate_matches_known_hash_good(certificate, known_hash):
    hash = hashlib.sha256(certificate).hexdigest() # GOOD
    return hash == known_hash
```
- Bad example (`weak_password_hashing_bad.py`):
```
import hashlib

def get_password_hash(password: str, salt: str):
    return hashlib.sha256(password + salt).hexdigest() # BAD
```
- Good example (`weak_password_hashing_good.py`):
```
from argon2 import PasswordHasher

def get_initial_hash(password: str):
    ph = PasswordHasher()
    return ph.hash(password) # GOOD

def check_password(password: str, known_hash):
    ph = PasswordHasher()
    return ph.verify(known_hash, password) # GOOD
```
