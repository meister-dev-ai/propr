Review focus: Assert statement tests the truth value of a literal constant

What to look for:
- An assert statement testing a literal constant value may exhibit different behavior when optimizations are enabled.
- In Python, assertions are not executed when optimizations are enabled. This may lead to unexpected behavior when assertions are used to the check validity of a piece of input.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`AssertLiteralConstant.py`):
```
def buy_bananas(n):
    if n > 500:
        assert False, "Too many bananas."
    send_order("bananas", n)

def buy_bananas_correct(n):
    if n > 500:
        raise AssertionError("Too many bananas")
    send_order("bananas", n)
```
