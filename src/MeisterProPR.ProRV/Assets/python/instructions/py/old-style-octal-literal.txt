Review focus: Confusing octal literal

What to look for:
- Octal literal with a leading 0 is easily misread as a decimal value
- Octal literals starting with 0 are easily misread as a decimal, particularly by those programmers who do not have a C or Java background. The new literal syntax for non-decimal numbers is more distinct and is thus less likely to be misunderstood.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`OldOctalLiteral.py`):
```
#Easily misread as x = 15
x = 015

#The extra 'o' alerts the reader that this is an octal literal
y = 0o15

#If this is a byte sized value then a hexadecimal might be clearer
y = 0x0d

#Or if it is a bit pattern then a binary value might be clearer
y = 0b1101
```
