Review focus: Unnecessary delete statement in function

What to look for:
- Using a 'delete' statement to delete a local variable is unnecessary, because the variable is deleted automatically when the function exits.
- Passing a local variable to a`del`statement results in that variable being removed from the local namespace. When exiting a function all local variables are deleted, so it is unnecessary to explicitly delete variables in such cases.
- Keep the check only when the changed code appears to introduce, remove, bypass, or misuse the risky behavior itself.
- Prefer explicit code evidence such as API calls, removed guards, changed data flow steps, changed arguments, or changed signatures over broad similarity.

Relevance threshold:
- Drop this check if the diff only changes naming, comments, formatting, or unrelated plumbing.
- Do not keep this check solely because the file belongs to a broad category like security, configuration, or API usage.
- This is direct-pattern focused. Keep it only when the diff shows the unsafe call, broken contract, or misuse pattern itself.

Reference examples:
- Reference example (`UnnecessaryDelete.py`):
```
def unnecessary_delete():
    x = get_some_object()
    do_calculation(x)
    del x                       # This del statement is unnecessary
```
